プルリクエストに必要な酔う
・二人以上が必要
・レビュアー、レビュイー、リポジトリを持っている人
・リポジトリをフォークして自分のリモートリポジトリにおいて、git cloneして作業する。
・レビュアー指定するにはレビュアーにレポジトリ権限を与えないとダメ
・プルリクエストの画面から行う方法もある。
・レビュアーがプルリクエストの内容をレビューする機能がある
・git push origin 作ったブランチ名(作っていないならmain)
・マージの仕方は三つある
・create merge commit トピックブランチに加えたすコミット全てがベースブランチにマージされる。またマージコミットというマージを記録するコミットもコミットされる。
・squash and mergeはトピックブランチで加えたコミットを一つのコミットにまとめたあとでマージします。マージした後はトピックブランチのコミット単位で細かい履歴の確認ができなくなる。が、ベースブランチの履歴がプルリクエスト単位で整理された形になるので見やすくなるそうだ。
・Rebase and mergeはブランチの枝分かれ道を変更するリベースという操作をしてからマージを行う。ベースブランチの細心のコミットから、ブランチが別れたようにすることができます。見た目、ベースブランチに何も変更が加えられていないように見える。
・リベース後、マージすると一直線のコミット履歴になる。
・スカッシュやリベートをするとコミット履歴は残るけど元のコミット履歴からハッシュ値が変わるので注意。つまりいつでも過去の状態に戻れる、という事ができなくなる。 
・プルとフェッチ、プル＝フェッチ＋マージ。フェッチだけだとワークツリーには影響しない。ただリモートにあるリポジトリをローカルに持ってくるだけだ。でもプルはワークツリーにも反映させる。
git pull origin(プル先のリモートリポジトリ) master(プルするブランチ名)
git fetch origin(プル先のリモートリポジトリ) 
ローカルレポジトリにまだ存在せずリモートリポジトリに存在するブランチの取得はgit fetch origin からgit checkout ブランチ名
